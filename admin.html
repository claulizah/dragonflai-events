🏗️ Arquitectura técnica

DragonflAI Events es un sitio de 3 capas, sin servidor propio ni framework — todo en HTML/CSS/JS plano.

1. Frontend — archivos HTML

  • dragonflai-v2.html — el generador: formulario, planes con IA, Save the Date, publicar invitación
  • invitacion.html — la página pública que ve cada invitado (RSVP, galería, itinerario)
  • index.html — landing/precios · terminos.html / aviso-de-privacidad.html — legales
  • admin.html — este panel

2. Backend

  • Supabase — Postgres (invitaciones, RSVPs, planes, perfiles), Auth, y Storage (fotos)
  • Cloudflare Worker (dragonflai-worker-v2) — proxy que oculta la API key de Anthropic y resuelve CORS
  • Netlify Edge Functions — invitation-og.js (vista previa al compartir en WhatsApp) y admin-usage.js (este panel)

3. IA

Claude Sonnet 4.5 vía API de Anthropic — genera el plan de evento y el Save the Date en llamadas paralelas.

Hosting y dominio

dragonflaievents.com en Netlify (Netlify DNS + SSL automático vía Let's Encrypt). Links de invitación: dragonflaievents.com/nombre-evento (los viejos /i/nombre-evento siguen funcionando).

🔑 Dónde vive cada credencial

Ninguna llave secreta vive en el código público del sitio — las que son sensibles están como variables de entorno en Netlify o Cloudflare. Los valores reales NO se muestran aquí, solo dónde encontrarlas/rotarlas.

CredencialTipoDónde vive
Supabase URL + anon keyPúblicaHardcodeada en dragonflai-v2.html / invitacion.html — segura de exponer, la protección real es RLS en Supabase
Supabase service_role keySecretaNetlify env var SUPABASE_SERVICE_ROLE_KEY · rotar en Supabase → Settings → API
Anthropic API keySecretaCloudflare Worker dragonflai-worker-v2 → Settings → Variables · rotar en console.anthropic.com
Anthropic Admin keySecretaNetlify env var ANTHROPIC_ADMIN_KEY · Claude Console → Settings → Admin API keys
Netlify API tokenSecretaNetlify env var NETLIFY_API_TOKEN · User settings → Applications
Cloudflare API tokenSecretaNetlify env var CF_API_TOKEN · Cloudflare → My Profile → API Tokens
Contraseña de este panelSecretaNetlify env var ADMIN_PASSWORD — cámbiala ahí si se te olvida o quieres rotarla

Regla general: si algo necesita permisos amplios (borrar cosas, ver todo, gastar dinero), va en una variable de entorno del backend — nunca en un archivo que suba al repositorio.

⚙️ Cómo hacer cambios

Cambios en la base de datos (tablas, políticas, columnas)

  • Claude te da un archivo .sql
  • Tú lo corres en Supabase → proyecto wpgdkfdoecohoowwoywx → SQL Editor → Run
  • Le pasas a Claude el resultado (texto o captura) para confirmar

Cambios en el código del sitio (HTML/JS)

  • Claude edita el archivo directamente y te lo comparte para descargar
  • Tú lo subes a GitHub (reemplazando el archivo existente) — Netlify hace deploy automático en segundos
  • Verifica en Netlify → Deploys que el último build sea el correcto antes de probar

Cambios en el Worker de IA

  • Cloudflare Dashboard → Workers & Pages → dragonflai-worker-v2
  • Editar código o variables de entorno ahí directamente

Variables de entorno nuevas

  • Netlify → Site settings → Environment variables → Add a variable
  • Marca el scope como "Functions" para que las Edge Functions puedan leerlas
  • Requiere un nuevo deploy para tomar efecto
